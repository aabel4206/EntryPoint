# Frontend Design

## Overview

The EntryPoint frontend will be built with React. The interface should be simple, clean, and easy for new students to navigate.

The frontend will provide separate views for students and administrators.

---

## Main Pages

### Landing Page

Purpose:

Introduce EntryPoint and explain its purpose.

Content:

- Project name
- Short description
- Login button
- Register button

---

### Register Page

Purpose:

Allow new students to create an account.

Fields:

- First name
- Last name
- Email
- Password
- Student type
- Major
- International student status

---

### Login Page

Purpose:

Allow users to log in.

Fields:

- Email
- Password

---

### Student Dashboard

Purpose:

Provide the main student homepage after login.

Content:

- Welcome message
- Checklist progress summary
- Important updates
- Quick links to resources

---

### Onboarding Checklist Page

Purpose:

Display personalized onboarding tasks.

Features:

- View checklist tasks
- Mark tasks complete
- Filter by category
- View task priority
- Open related resource links

---

### Resource Directory Page

Purpose:

Display categorized Texas State resources.

Features:

- View resources by category
- Search resources
- Filter resources
- Open official resource links

---

### Resource Detail Page

Purpose:

Display detailed information about one resource.

Content:

- Resource title
- Description
- Category
- Official URL
- Last reviewed date

---

## Admin Pages

### Admin Dashboard

Purpose:

Allow administrators to manage EntryPoint content.

Content:

- Resource management links
- Checklist management links
- Webpage monitoring results

---

### Manage Resources Page

Purpose:

Allow administrators to create, edit, and delete resources.

Features:

- Add resource
- Edit resource
- Delete resource
- Assign category

---

### Manage Checklists Page

Purpose:

Allow administrators to create and edit onboarding checklist templates and tasks.

Features:

- Create checklist template
- Edit checklist template
- Add checklist tasks
- Connect tasks to resources

---

### Monitoring Page

Purpose:

Allow administrators to review webpage monitoring results.

Content:

- Monitored webpage list
- Detected changes
- Change summaries
- Importance levels
- Review status

---

## Navigation

### Student Navigation

- Dashboard
- Checklist
- Resources
- Updates
- Profile
- Logout

### Admin Navigation

- Admin Dashboard
- Resources
- Checklists
- Monitoring
- Logout

---

## UI Design Goals

- Simple layout
- Clear navigation
- Minimal clutter
- Mobile-friendly layout
- Accessible colors and readable text
- Clear checklist progress indicators

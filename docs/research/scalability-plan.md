# Scalability Plan

## Current Pilot Domain

EntryPoint currently uses transportation onboarding as the pilot domain.

## Framework Idea

The project is designed as a personalized onboarding framework, not just a transportation app. Transportation is the first domain used to test the framework.

## Scaling to Other Domains

The same structure can later support other onboarding areas, such as accommodation, healthcare, academic advising, and employment.

## Example: Accommodation

To support accommodation, the system would add:

- Housing resource categories
- Accommodation-specific resources
- Housing onboarding templates
- Monitored housing webpages

Examples include:

- Residence life
- Lease information
- Move-in requirements
- Utilities
- Roommate guidance

## What Stays the Same

The following parts remain reusable:

- Student profile model
- Resource category model
- Resource model
- Checklist generation structure
- Monitoring approach
- Admin review workflow

## What Changes

Only the domain-specific resources, templates, and monitored URLs need to change.
